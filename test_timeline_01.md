# test_timeline_01

PauseInk の手動確認用メモ。自動 test の代わりではなく、GUI 上の操作確認を短時間で抜け漏れなく回すための手順書です。

- 対象ブランチ: `develop`
- 想定 OS: Windows / macOS / Linux 共通
- 修飾キー: Windows / Linux は `Ctrl`、macOS は `Cmd`
- 前提: `cargo run -p pauseink-app` で起動できること
- 注意: v1.0 の outline 編集では `object 選択` と `group 選択` は同時保持しない。どちらか一方へ切り替わる。

## 1. オブジェクト選択・編集・グループ化の確認

### 1.1 準備

1. 新規 project を開き、再生を止めたまま 1 ページ目に次の 7 object を作る。
2. まず、互いに重ならない位置へ 4 つの目印を書く。
   例: 左上 `A`、右上 `B`、左下 `C`、右下 `D`
3. 次に、中央付近へ少し重なるように 3 つの目印を書く。
   例: `E`、`F`、`G`
4. 下部タブを `オブジェクト一覧` にする。

期待結果:

- object が合計 7 件見える
- `A`〜`D` は単体確認用、`E`〜`G` は重なり確認用として使える

### 1.2 単一選択

1. `A` に対応する object 行をクリックする。
2. 右ペインで色または太さを大きく変える。
3. 続けて `B` の行を通常クリックし、同じく色または太さを変える。

期待結果:

- 最初の変更は `A` だけに出る
- `B` を通常クリックした時点で選択は `B` に置き換わる
- `グループ化` は 1 object 選択中のままでは有効にならない

### 1.3 複数選択の追加・解除

1. `A` を選択した状態で、修飾キーを押しながら `B` をクリックする。
2. 色を変える。
3. 同じ修飾キーを押しながら `B` をもう一度クリックする。
4. 太さを変える。

期待結果:

- 1 回目の変更は `A` と `B` の両方に出る
- 2 回目のクリックで `B` だけ selection から外れる
- 2 回目の変更は残った選択対象だけに出る

### 1.4 グループ化

1. 修飾キーを押しながら `C` と `D` を選ぶ。
2. `グループ化` を押す。
3. 新しく出た group 行をクリックする。
4. 右ペインで色または太さを変える。

期待結果:

- `C` と `D` の group 行が 1 件増える
- group 行を選ぶと member object をまとめて操作できる
- group 行選択での style 変更は `C` と `D` の両方へ入る
- `A`、`B`、`E`、`F`、`G` は変わらない

### 1.5 グループ解除

1. 直前に作った group 行を選ぶ。
2. `グループ解除` を押す。

期待結果:

- group 行が消える
- `C` と `D` は ungrouped object として一覧へ戻る
- 解除前に入れた色や太さは残る

### 1.6 object 単位の z-order

1. 中央の `E`、`F`、`G` のうち 1 つを選ぶ。
2. 必要なら分かりやすい色へ変えて、どれを動かしているか識別できるようにする。
3. `背面へ` を押す。
4. 続けて `前面へ` を押す。
5. さらに `一つ後ろ`、`一つ前` も試す。

期待結果:

- 重なり順だけが変わる
- `背面へ` は完全に後ろ、`前面へ` は完全に前へ移る
- `一つ後ろ` / `一つ前` は 1 段ずつだけ変わる
- object 自体の形、色、page 所属は変わらない

### 1.7 group 単位の z-order

1. 修飾キーを押しながら `E` と `F` を選び、`グループ化` する。
2. できた group 行を選ぶ。
3. `背面へ` と `前面へ` を試す。

期待結果:

- `E` と `F` が 1 つの塊として `G` に対して前後する
- `E` と `F` の内部順序は崩れない
- `G` は group 外 object のまま残る

### 1.8 object 複数選択への entrance 一括適用

1. `E` と `F` の group 行を選ぶ。
2. `グループ解除` して、`E` と `F` を ungrouped object に戻す。
3. 修飾キーを押しながら `E` と `F` を複数選択する。
4. 右ペインで `出現` を `なぞりがき` にし、`出現速度` を極端に上げる。
5. 再生して少し確認し、停止する。

期待結果:

- `E` と `F` だけが同じ entrance 設定へ変わる
- `A`、`B`、`C`、`D`、`G` は従来設定のまま
- 再生時、`E` と `F` だけが同じ entrance 挙動になる

### 1.9 object 選択と group 選択の切り替え

1. `C` と `D` を複数選択して `グループ化` する。
2. できた group 行を選ぶ。
3. 修飾キーを押しながら、ungrouped の `A` をクリックする。
4. 色または太さを変える。

期待結果:

- group 行を選んだ時点では group selection だけが有効になる
- object 行を修飾キー付きでクリックしても、group と object の mixed selection にはならない
- `A` をクリックした時点で group selection は外れ、object selection 側へ切り替わる
- 変更はその時点で選ばれている object 側にだけ入る

### 1.10 undo / redo

1. ここまでの操作を数手進めた状態で `Ctrl+Z` または `Cmd+Z` を数回押す。
2. つづけて `Ctrl+Shift+Z` または `Cmd+Shift+Z`、必要なら `Ctrl+Y` も試す。

期待結果:

- group 化、group 解除、色変更、entrance 変更、z-order 変更が逆順で戻る
- redo で同じ順に戻る
- undo / redo 後も selection が壊れず、操作不能な ghost selection にならない
- ungroup を undo した時は group selection に戻り、group 化を undo した時は member object selection に戻る

### 1.11 保存と再読込

1. いまの project を保存する。
2. アプリを閉じるか、別 project を挟んでから再度その `.pauseink` を開く。

期待結果:

- group の member 関係が残る
- z-order の見た目が残る
- style と entrance の変更が残る
- 選択状態そのものは復元されなくてよい

## 2. V1-09〜V1-14 の簡易確認

### 2.1 V1-09 テンプレート font 切替 crash fix

1. テンプレート表示を出したまま、font dropdown を何度か切り替える。

期待結果:

- 落ちない
- テンプレート preview がその場で更新される

### 2.2 V1-10 テンプレート入力欄の複数行化

1. テンプレート入力欄へ 2 行以上入力する。
2. `Enter` で改行する。
3. 入力欄の右下をドラッグして高さを変える。

期待結果:

- 初期状態で 2 行分の高さがある
- 改行がそのまま template preview に反映される
- 高さ変更で入力欄だけが伸びる

### 2.3 V1-11 パネル幅追従

1. 左右ペインを広げたり狭めたりする。
2. シークバー、テンプレート入力欄、preset 名入力欄を見る。

期待結果:

- 主要入力欄が panel 幅に追従して自然に広がる
- 余白やボタン列を不自然に潰さない

### 2.4 V1-12 template slot UI removal

1. 左ペインの template 周辺を見る。

期待結果:

- `前スロット` / `次スロット` ボタンが無い
- `slot: 3/8` のような状態表示も無い

### 2.5 V1-13 Esc cancel

1. 設定やテンプレート詳細などの window を開いて `Esc` を押す。
2. window を閉じた後、template preview や guide を出して `Esc` を押す。
3. 文字入力欄に focus を置いたまま `Esc` を押す。

期待結果:

- 先に window が閉じる
- window が無い時だけ template preview / guide が解除される
- text 入力中は global cancel が割り込まない

### 2.6 V1-14 metrics-based template alignment

1. テンプレート文字列に `x y A . VA` を入れる。
2. `英字倍率` と `句読点倍率` を下げる。
3. 必要なら font を変えて見比べる。

期待結果:

- `x` より `y` の方が少し下へ伸びる
- 小さくした英字や句読点が上に浮かず、下側へ寄る
- `VA` の横間隔が fixed-width のように崩れない
